# QR Code Create - 要件定義書

## 1. プロジェクト概要

### 1.1 プロダクト名
QR Code Create（仮称）

### 1.2 コンセプト
URLからQRコードを生成する際に、マーケティングに必要なUTMパラメータや装飾を「忘れずに」設定できるWebサービス。
単なるQRコード生成ではなく、マーケティング効果を最大化するためのガイド付きQRコード生成ツール。

### 1.3 背景・課題
- QRコードを生成するサイトは多数存在するが、UTMパラメータの付与を促すものは少ない
- UTMパラメータや説明文を付け忘れてしまい、後からマーケティング効果の計測ができないケースが多い
- QRコードの見た目にこだわりたいが、装飾オプションが少ないサービスが多い

### 1.4 ゴール
- QRコード生成時にUTMパラメータの設定をガイドし、付け忘れを防止する
- UTMパラメータに詳しくないユーザーでも適切な値を選べるようにする
- マーケティング効果を高める装飾（フレーム、キャプション、ロゴ等）を簡単に適用できる

---

## 2. ターゲットユーザー

### 2.1 主要ターゲット
- **事業主**: 自社サービスのプロモーションにQRコードを利用する人
- **イベント主催者**: チラシ・ポスター・名刺等にQRコードを掲載する人

### 2.2 ユーザー特性
- マーケティングの専門家ではない
- UTMパラメータという用語を知らない、または聞いたことがあるが何を設定すべきかわからない
- QRコードの見た目にもこだわりたい
- 無料で手軽に使いたい（ログインなし）

## 3. 技術スタック

| 項目 | 技術 |
|---|---|
| フレームワーク | Next.js (App Router) |
| スタイリング | CSS Modules + CSS Custom Properties |
| デプロイ先 | Vercel |
| バックエンド | 不要（フロントエンドのみで完結） |
| DB | 不要 |
| 認証 | 不要 |

### 3.1 技術方針
- QRコード生成はクライアントサイドで完結する（ライブラリ使用）
- 画像のダウンロードもクライアントサイドで行う
- サーバーサイドの処理は不要（静的サイトに近い構成）

## 4. 機能要件

### 4.1 MVP（Phase 1）機能一覧

#### F-001: URL入力とUTMパラメータ設定（コア機能）
- **概要**: QRコードに埋め込むURLを入力し、UTMパラメータを付与する
- **詳細**:
  - URL入力フィールド（必須）
  - UTMパラメータ入力セクション（すべて任意だが、設定を強く推奨するUIとする）
    - `utm_source`: トラフィックの参照元（例: google, facebook, newsletter）
      - プルダウンでおすすめ選択肢を提示 + 自由入力可
    - `utm_medium`: マーケティング媒体（例: cpc, email, social, qr, print）
      - プルダウンでおすすめ選択肢を提示 + 自由入力可
    - `utm_campaign`: キャンペーン名（例: spring_sale_2026, event_open_day）
      - 自由入力 + 命名規則のガイダンス表示
    - `utm_term`: 検索キーワード（任意）
      - 用途の説明付き
    - `utm_content`: コンテンツの識別（任意）
      - 用途の説明付き（A/Bテスト等での使い分け）
  - 各UTMパラメータにはツールチップ/ヘルプテキストで「これは何か」「どう設定すべきか」を説明する
  - UTMパラメータ未設定時には「UTMパラメータを設定すると、Google Analyticsで効果測定ができます」等の注意メッセージを表示する
  - 最終的に生成されるURL（UTMパラメータ付き）をプレビュー表示する

#### F-002: UTMパラメータのガイド機能
- **概要**: UTMパラメータに詳しくないユーザーでも適切に設定できるガイダンス
- **詳細**:
  - 各パラメータの説明（ツールチップ/インライン説明文）
    - 「utm_sourceとは？」→「QRコードを掲載する場所（チラシ、名刺、ポスターなど）を指定します」
    - 「utm_mediumとは？」→「どのような手段でユーザーに届けるかを指定します。QRコードの場合は"qr"がおすすめです」
    - 等、初心者向けのわかりやすい説明
  - utm_source / utm_medium にはシステム側で用意したおすすめ選択肢をプルダウンで提示
    - utm_source例: `flyer`（チラシ）, `poster`（ポスター）, `business_card`（名刺）, `brochure`（パンフレット）, `banner`（看板）, `receipt`（レシート）, `product_package`（製品パッケージ）
    - utm_medium例: `qr`（QRコード）, `print`（印刷物）
  - utm_campaignの命名ガイダンス
    - 「日付や内容を含めると管理しやすくなります（例: spring_sale_2026）」
  - QRコードの配布用途に応じた設定例の提示
    - 例: 「名刺に印刷する場合」→ source: business_card, medium: qr

#### F-003: QRコード生成・プレビュー
- **概要**: 入力情報に基づいてQRコードをリアルタイムにプレビュー表示する
- **詳細**:
  - URL + UTMパラメータからQRコードを生成
  - 入力内容の変更に応じてリアルタイムでプレビューが更新される
  - QRコードの読み取り可否をシステム側で検証する（ロゴが大きすぎて読めない等の防止）

#### F-004: QRコードの装飾機能
- **概要**: QRコードの見た目をカスタマイズする
- **詳細**:
  - **中央ロゴ画像**
    - ユーザーが画像をアップロードできる
    - アップロードされた画像をQRコード中央に配置
    - ロゴサイズの調整（QRコードの読み取りを阻害しない範囲）
    - 対応フォーマット: PNG, JPG, SVG
  - **色のカスタマイズ**
    - 前景色（QRコードのドット色）の変更
    - 背景色の変更
    - コントラスト不足の場合に警告表示（読み取りにくくなることを防止）
  - **フレーム（枠）**
    - シンプル枠（線のみ）
    - 角丸枠
    - 装飾枠（額縁風、花柄、幾何学模様等）
    - ターゲット別テーマ枠
      - 女性向け（エレガント、パステル系）
      - 子供向け（カラフル、ポップ）
      - ビジネス向け（シンプル、フォーマル）
      - イベント向け（華やか、季節感）
    - 枠の色の変更
  - **キャプション（QRコード下部のテキスト）**
    - 自由入力テキスト
    - テンプレート選択肢（「スキャンして詳細を見る」「こちらからアクセス」等）
    - 適正文字数のガイダンス表示（印刷サイズに応じた推奨文字数）
    - フォントサイズ調整
  - **装飾プリセット**
    - 「シンプル」「ビジネス」「カジュアル」「エレガント」「ポップ」等のプリセットを用意
    - プリセット選択後に個別カスタマイズも可能

#### F-005: QRコード画像のダウンロード
- **概要**: 生成したQRコード（装飾含む）を画像としてダウンロードする
- **詳細**:
  - 対応フォーマット: PNG, SVG, PDF
  - 装飾（フレーム、キャプション、ロゴ）を含めた状態でダウンロード
  - 印刷用途を考慮した高解像度出力オプション

## 5. 非機能要件

### 5.1 パフォーマンス
- QRコードのプレビュー更新は入力変更後500ms以内
- ダウンロード処理は3秒以内に完了

### 5.2 対応環境
- モダンブラウザ（Chrome, Firefox, Safari, Edge の最新2バージョン）
- レスポンシブデザイン（PC / タブレット / スマートフォン対応）

### 5.3 アクセシビリティ
- WCAG 2.1 Level AA準拠を目標
- キーボード操作での全機能利用
- スクリーンリーダー対応

### 5.4 セキュリティ
- ユーザーがアップロードしたロゴ画像はサーバーに保存しない（クライアントサイドで処理）
- XSS対策（URL入力のサニタイズ）
- 入力URLの形式バリデーション

### 5.5 国際化
- Phase 1は日本語UIのみ
- 将来的に英語対応を検討

## 7. 将来構想（Phase 2以降）

以下は Phase 1（MVP）には含めないが、将来的に検討する機能。

### 7.1 ユーザーアカウント機能
- ログイン/サインアップ（メール、Google OAuth等）
- 生成したQRコードの履歴保存・管理
- UTMパラメータテンプレートの保存

### 7.2 短縮URL機能（有料オプション候補）
- システム独自の短縮URLを発行し、QRコードに埋め込む
- 短縮URLには有効期限を設定（発行後1年間）
- 短縮URL経由のアクセス数（スキャン数）をダッシュボードで確認可能
- 日別・時間帯別のアクセス推移
- リファラー・デバイス情報の簡易分析

### 7.3 有料プラン
- 短縮URL + スキャン計測機能
- カスタムドメイン短縮URL
- 高度な装飾テンプレート
- ブランドキットの保存（ロゴ・色の保存）

### 7.4 多言語対応
- 英語UI対応
- その他言語の検討

## 8. 用語集

| 用語 | 説明 |
|---|---|
| UTMパラメータ | URLの末尾に付与するトラッキング用パラメータ。Google Analyticsなどでトラフィック元を分析するために使用される。utm_source, utm_medium, utm_campaign, utm_term, utm_content の5種類がある |
| utm_source | トラフィックの参照元。QRコードの場合、QRコードを掲載した媒体（チラシ、名刺等）を指定する |
| utm_medium | マーケティングの媒体種別。QRコードの場合は"qr"や"print"を使用する |
| utm_campaign | キャンペーン名。プロモーション施策ごとに一意の名前を付けて管理する |
| utm_term | 検索キーワード。主にリスティング広告で使用される |
| utm_content | 広告コンテンツの識別子。同一キャンペーン内でA/Bテストを行う場合などに使用する |
| QRコード | Quick Responseコードの略。マトリックス型2次元バーコードの一種 |
| フレーム | QRコードの周囲を囲む装飾枠 |
| キャプション | QRコードの下部に表示する説明テキスト |
