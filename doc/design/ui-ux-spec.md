**操作フロー**:
1. URL入力完了後に自動展開（プログレッシブディスクロージャー）
2. 損失回避メッセージが最初に目に入る
3. utm_medium は「QRコード (qr)」がデフォルト選択済み（Default Effect）
4. utm_source はプルダウンからおすすめ値を選択（「その他」で自由入力も可）
5. utm_campaign は自由入力 + 命名ガイダンス
6. utm_term / utm_content は「詳細設定」として折りたたみ
7. 設定例カードから一括設定も可能
8. 各フィールド変更時にGA4イベント発火

**ツールチップ（`[?]` アイコン）の内容**:

| フィールド | ツールチップ内容 |
|---|---|
| utm_source | 「QRコードを掲載する場所（チラシ、名刺、ポスターなど）を指定します。Google Analyticsで"参照元"として表示されます。」 |
| utm_medium | 「どのような手段でユーザーに届けるかを指定します。QRコードの場合は"qr"がおすすめです。」 |
| utm_campaign | 「キャンペーンや施策の名前です。後から見て何のことかわかる名前を付けましょう。日付を含めると管理しやすくなります。」 |
| utm_term | 「検索キーワードを指定します。QRコードの場合はあまり使いませんが、特定のキーワードに紐づけたい場合に使います。」 |
| utm_content | 「同じキャンペーンで複数のQRコードを使い分ける場合に使います。例: ヘッダーとフッターに異なるQRコードを配置する場合。」 |

## 3. プログレッシブディスクロージャー

[マーケティング戦略 セクション7](../marketing-strategy.md) のCRO施策に基づく段階的UI表示。

### 3.1 ステップ遷移ルール

| 現在のステップ | 次のステップへの条件 | 表示変化 |
|---|---|---|
| ステップ1（URL入力） | 有効なURLが入力された | ステップ2が自動展開。QRプレビューが表示される |
| ステップ2（UTM設定） | 常にアクセス可能（URLが有効なら） | ステップ3のタブが有効化 |
| ステップ3（装飾設定） | 常にアクセス可能（URLが有効なら） | ダウンロードボタンが有効化 |
| ダウンロード | QRコードが生成済み | ダウンロード実行可能 |

- 各ステップは折りたたみ可能（アコーディオン形式）
- 一度展開したステップは自由に開閉できる
- ProgressBarは常に表示

### 3.2 ProgressBar の完了度算出

完了度は3段階（33% / 67% / 100%）で算出する。ダウンロードはQRコード完成後のアクションであり、完成度には含めない。

```typescript
function getCompletionPercent(state: QrState): number {
  let steps = 0;

  // ステップ1: URL入力 (+33%)
  if (state.isUrlValid) steps += 1;

  // ステップ2: UTM設定 (+33%)
  const hasUtm = /* source/medium/campaign/term/contentのいずれかが入力済み */;
  if (hasUtm) steps += 1;

  // ステップ3: 装飾設定 (+34%)
  const hasDecoration = /* 色変更/フレーム/キャプション/ロゴのいずれかが設定済み */;
  if (hasDecoration) steps += 1;

  return Math.round((steps / 3) * 100);
}
```

### 3.3 ProgressBar のメッセージ

| 完了度 | メッセージ |
|---|---|
| 0% | URLを入力してQRコードを作成しましょう |
| 33% | UTMパラメータを設定するとマーケティング効果UP！ |
| 67% | 装飾でQRコードをもっと魅力的に |
| 100% | QRコードが完成しました！ |

Zeigarnik Effect（未完了タスクが記憶に残る効果）を活用し、常に「あと少し」感を演出する。

## 6. レスポンシブデザイン方針

### 6.1 ブレークポイント別レイアウト

| コンポーネント | スマートフォン | タブレット | デスクトップ |
|---|---|---|---|
| ヘッダーナビ | ハンバーガーメニュー | ハンバーガーメニュー | 横並びリンク |
| QrGenerator | 1カラム | 1カラム | 2カラム（入力/プレビュー） |
| QRプレビュー | スティッキー（上部固定） | スティッキー（上部固定） | 右カラム |
| 特徴セクション | 1カラム | 2カラム | 3カラム |
| 用途別カード | 2カラム | 3カラム | 4カラム |
| フレーム選択 | 2カラムグリッド | 3カラムグリッド | 4カラムグリッド |
| 装飾プリセット | 2カラムグリッド | 3カラムグリッド | 5カラム横並び |

### 6.2 スマートフォン固有の配慮

- タップターゲットは最低44x44px
- フォーム要素は十分な余白を確保（誤タップ防止）
- QRプレビューはスクロール追従（position: sticky）
- ダウンロードボタンは画面下部に固定表示（固定フッター）
- ツールチップはモーダル表示に切り替え

## 7. アクセシビリティ

[要件定義書 5.3](../requirements.md) に基づきWCAG 2.1 Level AA準拠を目標とする。

### 7.1 キーボード操作

| 操作 | キー | 対象 |
|---|---|---|
| フォーカス移動 | Tab / Shift+Tab | 全インタラクティブ要素 |
| 選択/実行 | Enter / Space | ボタン、リンク、チェックボックス |
| ドロップダウン操作 | ↑↓ | Select コンポーネント |
| タブ切り替え | ←→ | Tabs コンポーネント |
| モーダル閉じる | Escape | ツールチップ、モバイルナビ |
| 色入力 | 直接テキスト入力 | ColorPicker（キーボードで色コード入力可能） |

### 7.2 ARIA属性

```html
<!-- ProgressBar -->
<div role="progressbar" aria-valuenow="50" aria-valuemin="0" aria-valuemax="100"
     aria-label="QRコード作成の進捗">

<!-- ツールチップ -->
<button aria-describedby="tooltip-utm-source">
  utm_source <span>?</span>
</button>
<div id="tooltip-utm-source" role="tooltip">...</div>

<!-- タブ -->
<div role="tablist" aria-label="装飾設定">
  <button role="tab" aria-selected="true" aria-controls="panel-preset">プリセット</button>
  <button role="tab" aria-selected="false" aria-controls="panel-logo">ロゴ</button>
</div>
<div role="tabpanel" id="panel-preset" aria-labelledby="tab-preset">...</div>

<!-- アコーディオン（ステップ） -->
<button aria-expanded="true" aria-controls="step2-content">
  ステップ2: UTMパラメータ設定
</button>
<div id="step2-content" role="region">...</div>

<!-- カラーピッカー -->
<label for="fg-color">前景色</label>
<input id="fg-color" type="color" aria-label="QRコードの前景色を選択">
<input type="text" aria-label="カラーコードを直接入力" pattern="#[0-9a-fA-F]{6}">

<!-- エラーメッセージ -->
<input aria-invalid="true" aria-describedby="url-error">
<span id="url-error" role="alert">有効なURLを入力してください</span>
```

### 7.3 色のコントラスト

- テキストと背景のコントラスト比は4.5:1以上（WCAG AA基準）
- 大きいテキスト（18px bold以上）は3:1以上
- フォーカスリングの視認性を確保
- QRコードのコントラスト警告を表示（3:1未満で警告）

### 7.4 スクリーンリーダー対応

- QRコードプレビュー画像に適切な `alt` テキスト: 「{URL}のQRコードプレビュー」
- 装飾変更時にライブリージョンで状態を通知: `aria-live="polite"`
- ダウンロード完了時に通知: `aria-live="assertive"`
- フォームのラベルとエラーメッセージを関連付け

## 8. マイクロインタラクション

### 8.1 QRコード更新アニメーション

- 入力変更後のQRコード再生成時、フェードイン/アウトで切り替え
- `transition: opacity 200ms ease`
- 生成中はスケルトン表示

### 8.2 ProgressBar アニメーション

- ステップ完了時にバーがスムーズに伸びる
- `transition: width 500ms ease-out`
- 完了時にパルスアニメーション

### 8.3 ステップ展開アニメーション

- アコーディオンの開閉は `max-height` + `opacity` でスムーズに
- `transition: max-height 300ms ease, opacity 200ms ease`

### 8.4 ボタンフィードバック

- ホバー: 背景色が暗くなる + 微小な上移動（`transform: translateY(-1px)`）
- クリック: 押し込みアニメーション（`transform: scale(0.98)`）
- ダウンロード完了: チェックマークアニメーション

### 8.5 コピーボタン

- クリック時: 「コピーしました」のトースト表示（2秒後に自動消去）
- アイコンがクリップボード → チェックマークに一時的に変化

## 9. エラー・警告表示パターン

### 9.1 バリデーションエラー

```
赤色ボーダー + エラーメッセージ（入力フィールド直下）
```

| 対象 | エラー条件 | メッセージ |
|---|---|---|
| URL | 空白 | URLを入力してください |
| URL | 無効な形式 | 有効なURLを入力してください（例: https://example.com） |
| URL | 長すぎる（2048文字超） | URLが長すぎます。短いURLを使用してください |
| ロゴファイル | サイズ超過 | ファイルサイズが大きすぎます（上限: 500KB） |
| ロゴファイル | 非対応形式 | PNG, JPG, SVG形式のファイルを選択してください |

### 9.2 警告メッセージ

```
黄色背景 + 警告アイコン + メッセージ
```

| 対象 | 警告条件 | メッセージ |
|---|---|---|
| コントラスト | 比率3:1未満 | コントラストが低いため、QRコードが読み取りにくくなる可能性があります |
| ロゴサイズ | 25%超 | ロゴが大きいため、QRコードの読み取りに影響する可能性があります |
| UTM未設定 | 0個 | UTMパラメータを設定すると、Google Analyticsで効果測定ができます |
| キャプション | 15文字超 | キャプションが長めです。印刷時に見切れる可能性があります |

### 9.3 成功メッセージ

```
緑色背景 + チェックアイコン + メッセージ
```

| 対象 | 条件 | メッセージ |
|---|---|---|
| URLコピー | コピー完了 | URLをコピーしました |
| ダウンロード | 完了 | QRコードをダウンロードしました |
